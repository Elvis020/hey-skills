# .gitignore Rules

## Step 1: Ask the user

Before touching `.gitignore`, ask:

> "Should `.codebase-indexer/` be committed to the repo (shared with the team) or gitignored (local to your machine only)?"

| Choice | When to use |
|--------|------------|
| **Committed (shared)** | Team project, docs help other contributors, you want docs to persist across machines |
| **Gitignored (local)** | Personal project, you don't want to add AI-generated files to the repo, or the repo has a policy against generated files |

## Step 2: Apply the choice

**If committed:** Do nothing to `.gitignore`. The index artifacts will be tracked by git normally.

**If gitignored:**
1. Read `.gitignore` if it exists.
2. If `.codebase-indexer/` is already listed — do nothing.
3. If `.gitignore` exists but lacks `.codebase-indexer/` — append:
   ```

   # Generated by codebase-indexer
   .codebase-indexer/
   ```
4. If `.gitignore` does not exist — create it with:
   ```
   # Generated by codebase-indexer
   .codebase-indexer/
   ```

**Never duplicate** — always check before writing.
